Concept Cards

Systematic one-page artifacts explaining fundamental NLP concepts with definitions, working principles, examples, and real-world applications.

A.1 Preprocessing

Sentence Segmentation

Definition

The process of splitting a large body of text into individual sentences based on boundary detection using punctuation marks and contextual cues.

Purpose

To break down unstructured text into manageable, semantically complete units that can be processed independently by downstream NLP tasks.

Working Principle

Uses rule-based heuristics (punctuation + capital letters), regex patterns, or ML models (CRF, BiLSTM) to identify boundaries. Handles abbreviations like "Dr.", "U.S.A." via exception lists.

Example
# Input "Dr. Smith visited the U.S.A. yesterday. He loved it!" # Output ["Dr. Smith visited the U.S.A. yesterday.", "He loved it!"]

Advantages

  • Foundation for all downstream NLP
  • Enables parallel processing
  • Isolates semantic units

Limitations

  • Ambiguity with abbreviations
  • Fails on informal text
  • Multi-language challenges
A.2 Preprocessing

Tokenization

Definition

Breaking text into smaller units (tokens): words, subwords, or characters that serve as basic input for NLP models.

Purpose

To convert continuous text into discrete, structured elements that algorithms can process, count, and analyze mathematically.

Working Principle

Word: Splits on whitespace/punctuation. Subword (BPE/WordPiece): Handles OOV by breaking into frequent units. Character: Each char is a token.

Example
# Word-level ["Tokenization", "is", "fundamental", "!"] # BPE subword ["Token", "ization", "is", "fundamental"]

Advantages

  • Standardizes model input
  • Subword handles OOV words
  • Enables vocabulary creation

Limitations

  • Word-level: large vocabulary
  • Character-level loses meaning
  • Language-dependent
A.3 Preprocessing

Case Normalization

Definition

Converting all text to a uniform case (typically lowercase) so the same word in different cases is treated identically.

Purpose

To reduce vocabulary size and prevent the model from treating "Apple", "apple", and "APPLE" as three different words.

Working Principle

Simple string transformation: text.lower(). Advanced: Truecasing preserves proper nouns; case can be a feature for POS tagging.

Example
# Input "The Apple CEO announced new Apple products." # Lowercase "the apple ceo announced new apple products" # Truecasing (preserves proper nouns) "The Apple CEO announced new Apple products"

Advantages

  • Reduces vocabulary size
  • Improves generalization
  • Computationally trivial O(n)

Limitations

  • Loses semantic info (US vs us)
  • NER may suffer
  • Sentiment intensity lost
A.4 Preprocessing

Stop-word Removal

Definition

Filtering out high-frequency, low-information words (articles, prepositions, conjunctions) that do not contribute significantly to semantic meaning.

Purpose

To reduce noise, decrease dimensionality, and help models focus on content-bearing words that carry actual meaning.

Working Principle

Uses predefined language-specific lists (NLTK: 179 English stopwords) or corpus-derived frequency thresholds. Domain-specific stopwords can be added.

Example
# Input "The quick brown fox jumps over the lazy dog" # After removal ["quick", "brown", "fox", "jumps", "lazy", "dog"]

Advantages

  • Reduces feature space 20-40%
  • Improves processing speed
  • Enhances signal-to-noise

Limitations

  • Removes negation ("not")
  • Phrase meaning lost
  • Domain-dependent
A.5 Preprocessing

Stemming

Definition

A heuristic process that chops off the ends of words to reduce them to their root form (stem), often resulting in non-dictionary words.

Purpose

To normalize different grammatical forms of a word to a common base, reducing vocabulary size and treating morphological variants as the same term.

Working Principle

Porter Stemmer: 5 phases of rule-based suffix stripping. Snowball: Improved Porter. Lancaster: More aggressive, faster.

Example
# Input ["running", "runs", "runner"] # Porter ["run", "run", "runner"] # Lancaster (aggressive) ["run", "run", "run"]

Advantages

  • Very fast O(n) per word
  • No dictionary required
  • Effective for IR systems

Limitations

  • Over-stemming: false matches
  • Under-stemming: misses forms
  • Produces non-words
A.6 Preprocessing

Lemmatization

Definition

A morphological analysis process that reduces words to their canonical dictionary form (lemma) using vocabulary and morphological analysis.

Purpose

To normalize words while preserving their valid linguistic form and part-of-speech, ensuring semantic accuracy in downstream tasks.

Working Principle

Uses lexical databases (WordNet) and POS tagging. "better" → "good" (JJR), "running" → "run" (verb) or "running" (noun). Requires POS tags for accuracy.

Example
# Input ["running", "better", "geese", "was"] # Output ["run", "good", "goose", "be"]

Advantages

  • Valid dictionary words
  • Handles irregular forms
  • Preserves semantic meaning

Limitations

  • Slower than stemming
  • Requires POS tags
  • Dictionary coverage gaps
A.7 Preprocessing

Noise Removal

Definition

The process of eliminating irrelevant, distracting, or malformed elements from text data that do not contribute to the intended NLP task.

Purpose

To clean raw text from artifacts like HTML tags, URLs, special characters, emojis, and encoding errors that can confuse models.

Working Principle

Regex-based: Remove URLs, HTML tags, special chars. Library-based: BeautifulSoup for HTML, emoji library for emoji conversion. Heuristic: Fix encoding.

Example
# Input "Check out https://example.com! Great deal!!!" # After noise removal "Check out Great deal"

Advantages

  • Improves data quality
  • Reduces model confusion
  • Standardizes input format

Limitations

  • May remove useful info (emojis)
  • Over-cleaning strips patterns
  • Domain-specific rules needed
A.8 Preprocessing

Text Cleaning

Definition

A comprehensive process encompassing multiple preprocessing steps to transform raw, unstructured text into clean, structured data ready for analysis.

Purpose

To ensure data quality, consistency, and compatibility with NLP algorithms by addressing all forms of textual irregularities systematically.

Working Principle

Pipeline: (1) Handle encoding → (2) Remove HTML → (3) Expand contractions → (4) Normalize case → (5) Remove noise → (6) Fix spelling → (7) Standardize whitespace.

Example
# Raw "I can't wait 4 the new iPhone!!! Visit: apple.com" # Cleaned "i cannot wait for the new iphone visit apple com"

Advantages

  • Systematic quality assurance
  • Reproducible across datasets
  • Customizable per domain

Limitations

  • Lossy process
  • Requires domain expertise
  • Order of operations matters
A.9 Feature Engineering

Bag of Words (BoW)

Definition

A text representation model describing the occurrence of words within a document, disregarding grammar and word order but keeping multiplicity.

Purpose

To convert text into a fixed-length numerical vector that ML algorithms can process, capturing word frequency as a feature.

Working Principle

Build vocabulary from all unique words. For each document, create a vector of size |V| where each position represents word count/frequency.

Example
# Corpus ["I love NLP", "NLP is great"] # Vocabulary: {I:0, love:1, NLP:2, is:3, great:4} # Doc1 vector [1, 1, 1, 0, 0] # Doc2 vector [0, 0, 1, 1, 1]

Advantages

  • Simple to implement
  • Effective for classification
  • Works with small datasets

Limitations

  • Ignores word order
  • High dimensionality
  • Sparse vectors
A.10 Feature Engineering

N-Grams

Definition

A contiguous sequence of n items (words or characters) from a given text sample, used to capture local word order and contextual information.

Purpose

To preserve some sequential information and local context that unigram models lose, improving representation quality.

Working Principle

Extract all n-length sliding windows from tokenized text. Unigram (n=1), Bigram (n=2), Trigram (n=3).

Example
# Text: "The quick brown fox" # Bigrams ["The quick", "quick brown", "brown fox"] # Trigrams ["The quick brown", "quick brown fox"]

Advantages

  • Captures local word order
  • Improves over BoW
  • Simple to implement

Limitations

  • Exponential feature growth
  • Data sparsity for high n
  • Misses long-range deps
A.11 Feature Engineering

Term Frequency (TF)

Definition

A statistical measure indicating how frequently a term occurs in a document, normalized by document length.

Purpose

To quantify the importance of a word within a specific document, assuming frequent words are more relevant to the topic.

Working Principle

Normalized TF: count(t,d) / total words in d. Log normalization: log(1 + count(t,d)) to dampen high frequencies.

Example
# Doc: "NLP is great. NLP is the future." (10 words) TF("NLP") = 2/10 = 0.2 TF("is") = 2/10 = 0.2 TF("future") = 1/10 = 0.1

Advantages

  • Simple and intuitive
  • Computationally efficient
  • Foundation for TF-IDF

Limitations

  • Favors common words
  • No corpus-level info
  • Sensitive to doc length
A.12 Feature Engineering

Inverse Document Frequency (IDF)

Definition

A measure of how rare or common a word is across an entire corpus, used to downweight frequent words and upweight rare, discriminative terms.

Purpose

To identify words that are distinctive to specific documents, helping distinguish between documents by penalizing corpus-common words.

Working Principle

IDF(t,D) = log(N / df(t)) where N = total documents, df(t) = documents containing term t. Higher IDF = more distinctive.

Example
# Corpus: 1000 documents "the" in 990 docs → IDF = log(1000/990) ≈ 0.01 "quantum" in 5 docs → IDF = log(1000/5) ≈ 5.3

Advantages

  • Identifies discriminative words
  • Reduces common word impact
  • Grounded in information theory

Limitations

  • Assumes term independence
  • Sensitive to corpus size
  • Cannot handle new docs
A.13 Feature Engineering

TF-IDF

Definition

A numerical statistic reflecting how important a word is to a document in a corpus, combining local importance (TF) with global rarity (IDF).

Purpose

To create a weighted representation highlighting words that are frequent in a specific document but rare across the corpus.

Working Principle

TF-IDF(t,d,D) = TF(t,d) × IDF(t,D). High TF-IDF = important and distinctive word for that document.

Example
# Document about "Machine Learning" "learning": TF=0.1, IDF=2.5 → TF-IDF = 0.25 "the": TF=0.15, IDF=0.01 → TF-IDF = 0.0015 "neural": TF=0.05, IDF=4.0 → TF-IDF = 0.20

Advantages

  • Balances local & global
  • Downweights common words
  • Intuitive and interpretable

Limitations

  • Ignores word order
  • Cannot capture synonymy
  • Static weights
A.14 Representation

One-Hot Encoding

Definition

A representation where each word is a binary vector with a single "1" at its index and "0" everywhere else.

Purpose

To convert categorical text data into a numerical format that ML models can process, treating each word as an independent category.

Working Principle

Vocabulary size = V. Each word is a V-dimensional vector. "apple" at index 5 → [0,0,0,0,0,1,0,0,...,0]. No semantic relationship.

Example
# Vocabulary: ["cat", "dog", "bird"] "cat" → [1, 0, 0] "dog" → [0, 1, 0] "bird" → [0, 0, 1] # |V| = 50,000 dims each!

Advantages

  • Simple and unambiguous
  • No ordinal relationship
  • Easy to implement

Limitations

  • Curse of dimensionality
  • No semantic meaning
  • Extremely sparse
A.15 Representation

Word Embeddings

Definition

Dense, low-dimensional vector representations of words where semantically similar words are mapped to nearby points in vector space.

Purpose

To capture semantic and syntactic relationships between words, enabling models to generalize based on meaning rather than just co-occurrence.

Working Principle

Neural networks learn to predict words from context. Similar words have similar vector directions. Analogies via vector arithmetic: king - man + woman ≈ queen.

Example
# "king" → [0.2, -0.5, 0.8, ..., 0.1] (300-dim) # Cosine similarities sim("king", "queen") ≈ 0.75 sim("king", "apple") ≈ 0.05 # Vector arithmetic "king" - "man" + "woman" ≈ "queen"

Advantages

  • Captures semantic relationships
  • Dense vectors (low dim)
  • Enables analogies

Limitations

  • Polysemy problem
  • Static representations
  • Requires large corpora
A.16 Representation

Word2Vec

Definition

A shallow neural network model (Google, 2013) that learns word embeddings by predicting words from their context (CBOW) or context from words (Skip-gram).

Purpose

To produce high-quality, dense word representations that capture semantic and syntactic relationships efficiently.

Working Principle

CBOW: Predicts target from context (faster, frequent words). Skip-gram: Predicts context from target (rare words). Uses negative sampling.

Example
# Training: "The cat sat on the mat" # Skip-gram (window=2) Input: "sat" → Predict: ["The", "cat", "on", "the"] # Result "cat" and "dog" have high cosine similarity

Advantages

  • Efficient on large corpora
  • Captures semantic + syntactic
  • Vector arithmetic analogies

Limitations

  • Single vector per word
  • No subword info
  • Context-independent
A.17 Representation

FastText

Definition

An extension of Word2Vec (Facebook AI, 2016) representing words as bags of character n-grams, enabling embeddings for rare and OOV words.

Purpose

To overcome Word2Vec's OOV limitation by incorporating subword information, effective for morphologically rich languages.

Working Principle

Each word = itself + character n-grams. "apple" → <ap, app, ppl, ple, le> + "apple". Vector = sum of all n-gram vectors.

Example
# Word: "university" Subwords (n=3): <un, uni, niv, ive, ver, ers, rsi, sit, ity, ty> Vector = sum(subword vectors) # OOV: "universities" Shares n-grams with "university" → can be represented!

Advantages

  • Handles OOV words
  • Good for rich morphology
  • Fast C++ implementation

Limitations

  • Worse on semantic tasks
  • Larger model size
  • Subword noise for short words
A.18 Representation

Contextual Embeddings

Definition

Dynamic word representations where the vector for a word changes based on its surrounding context, addressing the polysemy problem.

Purpose

To generate context-dependent representations capturing different meanings of the same word in different sentences.

Working Principle

ELMo: BiLSTM-based. BERT: Masked LM + next sentence prediction. GPT: Autoregressive Transformer. Context via self-attention.

Example
# Sentence 1 "I sat by the river bank." # Sentence 2 "I deposited money at the bank." # "bank" has DIFFERENT vectors! Cosine similarity between the two ≈ 0.3 (low)

Advantages

  • Solves polysemy
  • State-of-the-art performance
  • Transfer learning paradigm

Limitations

  • Computationally expensive
  • Large model sizes
  • Black-box nature
A.19 Language Models

Statistical Language Models

Definition

Language models that estimate the probability of word sequences using statistical methods based on observed frequencies in training corpora.

Purpose

To predict the next word in a sequence or evaluate the likelihood of given text, forming the foundation for speech recognition and MT.

Working Principle

N-gram models: P(w_n | w_1...w_{n-1}) ≈ count(w_1...w_n) / count(w_1...w_{n-1}). Smoothing: Laplace, Kneser-Ney for unseen n-grams.

Example
# Bigram model P("learning" | "machine") = count("machine learning") / count("machine") # If "machine learning" = 50, "machine" = 200 P = 50/200 = 0.25

Advantages

  • Simple and interpretable
  • Fast inference
  • Works with small data

Limitations

  • Data sparsity
  • No semantic understanding
  • Exponential storage growth
A.20 Language Models

Neural Language Models

Definition

Language models using neural networks (RNNs, LSTMs, GRUs) to learn distributed representations and predict word probabilities.

Purpose

To overcome data sparsity and generalization limitations of statistical n-gram models by learning continuous representations.

Working Principle

RNN/LSTM processes text sequentially, maintaining hidden state. Embedding layer converts words to dense vectors. Output: softmax over vocabulary.

Example
# Input: "The cat sat on the..." Embedding: "The" → [0.1, -0.3, ...] LSTM processes sequence, hidden state captures context Output: P("mat")=0.4, P("chair")=0.2, ...

Advantages

  • Long-range dependencies
  • Generalizes to unseen combos
  • Dense representations

Limitations

  • Sequential = slow training
  • Vanishing gradients
  • Limited context window
A.21 Language Models

Transformer-Based Representations

Definition

Deep learning architectures based on the Transformer mechanism (self-attention) that process entire sequences in parallel.

Purpose

To achieve state-of-the-art performance across NLP tasks by capturing global dependencies and enabling transfer learning.

Working Principle

Self-Attention: Attention(Q,K,V) = softmax(QK^T/√d_k)V. Multi-Head Attention captures different relationship types. Positional Encoding injects order.

Example
# "The animal didn't cross the street because it was too tired." Self-attention allows "it" to attend strongly to "animal" (not "street") BERT encodes this relationship in the contextualized embedding

Advantages

  • Parallel processing
  • Global dependencies
  • Transfer learning
  • Scalable to billions

Limitations

  • Quadratic O(n²) complexity
  • Massive compute needs
  • Environmental concerns
