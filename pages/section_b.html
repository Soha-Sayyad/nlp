Comparative Analysis

In-depth comparisons of NLP techniques across working mechanisms, computational complexity, strengths, weaknesses, and suitable applications.

Stemming vs Lemmatization

Criteria Stemming Lemmatization
Working Mechanism Applies rule-based algorithm for stripping off the suffixes such as Porter, Snowball, and Lancaster algorithms. Does not require the understanding of word meaning while applying rules. Lexical database such as WordNet is used for morphological analysis along with POS tag of the word.
Computational Complexity O(n) per word, very fast, does not require consulting any dictionary O(n × log|V|)
Strengths • Very fast
• No external dependencies
• Useful for information retrieval
• Smaller vocabulary 		• Linguistically correct
• Preserves semantics
• Works well with irregular words
• Better for semantic NLP applications
Weaknesses • Overstemming might cause mismatches
• Understemming might not work for related words
• Creates non-existing words in dictionary
• Does not understand semantics 		• Slow
• Requires POS tagger to be precise
• Lacks some domain-specific words in dictionary
• Hard to implement
Suitable Applications Search engines, information retrieval, spam filtering, keyword extraction Sentiment analysis, chatbots, machine translation, legal/clinical NLP

TF vs TF-IDF

Criteria Term Frequency (TF) TF-IDF
Working Mechanism • Calculates the number of times a term occurs in a document. The more occurrences, the higher the score. Combination of Term Frequency and Inverse Document Frequency.Makes common words across different documents less influential.
Computational Complexity O(|d|)
Scans only the current document. 		O(|d| + |D|)
Scans the document and requires corpus statistics.
Strengths • Easy to implement.
• Quick calculation process.
• Useful in frequency based applications. 		• Focuses on the most relevant words.
• Lowers the influence of common terms.
• Popular in information retrieval applications.
Weaknesses • Can't evaluate word importance.
• Assigns high scores to common words.
• Doesn't se corpus information. 		• Doesn't take into account word order and its context.
• Needs corpus statistics.
• Bad in semantic understanding.
Suitable Applications Keyword counting, logging information, document statistics. Search engines, classification, information retrieval.

Word2Vec vs FastText

Criteria Word2Vec FastText
Working Mechanism Learns word representations based on context words. Produces one vector for each word. Extension of Word2Vec that makes use of character level n-grams. Vector creation is done using both words and subwords.
Computational Complexity O(V × D)
Depending on the size of the vocabulary and embeddings. 		O((V + N-grams) × D)
Higher complexity because of the use of character n-grams.
Strengths • Semantic relationship learning is very good.
• Small size of the model.
• Fast training and inference. 		• Can handle out of vocabulary words.
• Handles well spelling variants.
• Good for morphologically rich languages.
Weaknesses • Poor handling of out of vocabulary words.
• Does not consider subwords.
• One vector for each word irrespective of variations. 		• High memory requirement.
• Longer training time.
• Complex model structure.
Suitable Applications Recommender systems, similarity tasks, news categorization. Spell correction, multi-lingual NLP, medical/technical texts.
